📌 项目名称

招聘网站岗位数据采集与薪资分析（Python / Playwright / Pandas）

📖 项目背景

招聘网站岗位数据通常存在以下问题：

页面采用 JS 渲染，传统 requests 难以获取数据

存在 基础反爬与滑块验证

薪资、城市等字段 格式混乱，无法直接分析

本项目从 数据采集 → 数据存储 → 数据清洗 → 数据分析 全流程出发，
构建一个可复用的招聘岗位数据分析项目。

🧱 项目整体流程
A. Playwright 自动化采集（51job）
   ↓
B. JSON 接口解析（岗位 / 公司 / 城市 / 薪资）
   ↓
C. MySQL 数据库存储
   ↓
D. Pandas 数据清洗与分析（Jupyter Notebook）

🔧 技术栈

Python 3

Playwright（处理 JS 渲染与反爬）

MySQL（结构化存储）

Pandas / NumPy（数据清洗与分析）

Jupyter Notebook（分析与可视化）

🚀 功能说明
1️⃣ 岗位数据采集

使用 Playwright 启动真实浏览器

通过 监听 XHR Response 获取真实岗位 JSON 数据

支持多页自动翻页（≈50 页）

支持多关键词搜索（如 Python / 数据）

采集字段包括：

岗位名称

公司名称

城市信息

薪资范围

学历要求

经验要求

技能标签

2️⃣ 数据库存储（MySQL）

设计结构化表结构

每条岗位作为一条记录写入数据库

支持后续长期采集与分析

3️⃣ 数据清洗与标准化（Pandas）
薪资字段处理

原始数据示例：

1-1.8万·13薪

40-60万/年

8千-1.2万

面议

处理方式：

使用正则解析区间

统一转换为 月薪（K）

计算 salary_low / salary_high / salary_avg

过滤不可量化数据

城市字段处理

原始字段可能包含：

上海·嘉定区

大连·甘井子区

浙江省

处理方式：

提取核心城市信息

构建城市等级（如一线 / 其他）

支持按城市维度进行统计分析

📊 分析示例

不同城市等级的薪资对比

学历 / 经验对薪资的影响

Python / 数据相关岗位的薪资分布

分析代码位于 analysis.ipynb

💡 项目收获

掌握 JS 渲染页面的数据采集思路

理解真实业务数据的 清洗与标准化流程

熟悉从 爬虫 → 数据库 → 分析 的完整数据链路

提升数据分析项目的工程化能力
